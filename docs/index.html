◈ Overview

gs-claude-config is a single Git repo that owns your entire ~/.claude/ configuration: slash commands, rich skill prompts, a global CLAUDE.md agent spec, and a settings.json template. Because ~/.claude/commands, ~/.claude/skills, and ~/.claude/CLAUDE.md are symlinks pointing here, every edit is instantly live in Claude Code — no restart required.

Two onboarding paths are available. chezmoi is the recommended path for new machines — a single command renders per-user templates, installs apps, and sets up fonts. The install script path is for existing setups or advanced users who want direct symlink control.

Key design principle

Everything in commands/, skills/, and CLAUDE.md is the real file. Edit here; Claude Code picks up changes immediately via transparent symlinks. settings.json is the only file not symlinked (it contains machine-specific keys).

◈ Installation

Recommended chezmoi — one command, full setup 
# Windows (PowerShell 7+)
winget install --id twpayne.chezmoi --scope user -e
chezmoi init --apply https://github.com/gsinvest017-ai/gs-claude-config.git
# macOS / Linux
brew install chezmoi   # or: sh -c "$(curl -fsLS get.chezmoi.io)"
chezmoi init --apply https://github.com/gsinvest017-ai/gs-claude-config.git

You'll be prompted for 7 values: name, email, githubUser, role, editor, installFonts, installCron.
To update later: cd ~/.local/share/chezmoi && git pull && chezmoi apply

Legacy install script — full symlink control 
git clone https://github.com/gsinvest017-ai/gs-claude-config.git ~/gs-claude-config
cd ~/gs-claude-config
./install.sh        # POSIX (Linux / macOS)
# or:
.\install.ps1       # Windows (PowerShell)

Idempotent. Backs up existing ~/.claude/{commands,skills,CLAUDE.md} before symlinking. Renders settings.template.json → ~/.claude/settings.json only if none exists.

After install — verify

ls -la ~/.claude/ | grep -E 'commands|skills|CLAUDE'
# should show 3 symlinks pointing to ~/gs-claude-config/

Day-to-day workflow

# Edit a skill (symlink is transparent):
$EDITOR ~/.claude/skills/safe-yolo/SKILL.md

# Commit & push:
cd ~/gs-claude-config && git add . && git commit -m "tweak skill" && git push

# Pull updates on another machine:
cd ~/gs-claude-config && git pull   # no restart needed

Adding a new skill

# Option 1 — use the built-in command:
/skill <describe what the skill should do>

# Option 2 — manual:
mkdir -p skills/<name>
# write skills/<name>/SKILL.md  (see any existing skill as template)
# optionally write commands/<name>.md  (slim entry-point)
git add skills/<name>/ commands/<name>.md && git commit -m "feat: add /<name> skill"

Night Shift — unattended /safe-yolo

Optional feature: a user-level cron fires at 00:00, walks repos in scripts/targets.conf, opens a fresh claude/nightly-YYYY-MM-DD branch, and runs /safe-yolo unattended until 06:00. Nothing is pushed; branches stay local for review. 

cp scripts/targets.conf.example scripts/targets.conf
$EDITOR scripts/targets.conf      # one repo path per line
scripts/install-cron.sh           # register 00:00 cron job

◈ Skills — 22 installed

Invoke any skill with /<skill-name> in a Claude Code session. Skills are rich prompt templates stored in skills/<name>/SKILL.md. Add args after the slash command name.

/apply-gs-templete
GS trading-portal の dark gold theme を現在 repo の dashboard UI に適用。CSS tokens、漸層、邊框を抽出。
/autogo
autogo dashboard が watch している画面の最新 OCR/segment 結果を会話 context に取り込む。-w <alias> でウィンドウ絞込み。
/cc-insights
~/.claude/projects/ の session JSONL を解析し、ツール頻度・ファイルホットスポット・繰り返しエラー・token ブラックホールを可視化。
/copy-commits-button
dashboard に「📋 今日の commits コピー」ボタンを注入。Markdown 形式でクリップボードに書き出し、作業グループに貼り付け可能。
/daily-summary
当日の git commits & 変更ファイルを繁体中文 Markdown 日報（YYYY-MM-DD-summary.md）にまとめる。カテゴリ分類 + リスク整理付き。
/git-tag
当日 commits を feature/fix/enhancement で分類し、各 milestone グループの最終 commit に annotated tag を付与。--apply で実行。
/language-tutor
1対1多言語家庭教師。CEFR A1–C2 評価、5モード（会話/単語/文法/ディクテーション/翻訳）、Windows TTS 発音サポート。
/lean-explain
Mathlib lemma の定義・signature・典型用法・corner case を繁体中文で解説。lean-lsp-mcp 経由でコンパイル可能な例を生成。
/lean-mil
MIL / TPiL ソクラテス式 Lean4 proof 指導。ヒントのみ提供、完全な proof は直接渡さない。3 ラウンド詰まったら方向性を示す。
/lean-prove
Lean4 proof goal を受け取り、LSP goal state を読みながら tactic を逐次推論、lake build が通るまで推進。sorry 使用禁止。
/one-button-launch
まだワンクリック起動がない repo に run.sh + run.ps1 を生成。install → build → migrate → start の一連フローを自動化。
/platform-compatible
Windows/Linux/macOS クロスプラットフォーム問題を監査・修正。パス区切り、EOL、ファイル名大文字小文字、CI matrix など 8 カテゴリ。
/prog-lang-tutor
repo を分析し、言語固有の idiom/構文/機構を抽出。知識バンクを構築し、Windows ポップアップで定期的に抜き打ちテスト。
/quant-researcher
量化策略研究員。理論推論 → 文献佐証 → コードバックテスト → 繁体中文レポートの 4 フェーズで戦略を産生。
/review-strategy
Jane Street レベルの戦略監査。Markdown 仕様の量化戦略を論理バグ・統計的落とし穴・サンプル外検証で審査し PASS/CONDITIONAL/FAIL 判定。
/safe-yolo
ノンストップ実行モード。2〜5 milestone に分解し、各完了後に commit + 進度ファイル更新。完了か完全行き詰まりまで継続。
/save-to-obsidian
会話を Obsidian vault に Markdown で保存。テーマ分類、frontmatter/tags 自動付与、obsidian:// URI でワンクリック開封。
/set-serve-eth
開発サーバーを LAN Ethernet IP にバインドし、Windows Defender Firewall / ufw でポートを開放（LocalSubnet のみ）。--apply で実行。
/test-plan
Unit / Integration / E2E の 3 層テスト計画を自動生成。フレームワーク自動検出、coverage matrix + 優先度順位付き。
/ui-compact
dashboard を no-scroll に圧縮。padding 縮小 → グラフ高さ調整 → grid 再配置 → accordion の順で軽から重へ段階的適用。
/update-doc
repo をスキャンし docs/index.html を生成・更新。GS dark gold テーマ、外部 CDN 不使用、MkDocs/Sphinx 等既存フレームワーク尊重。
/write-spec
repo を分析し、Claude が確実に遵守する CLAUDE.md を生成・更新。拆分対応（@import / nested）、update モードで人手コンテンツ保護。

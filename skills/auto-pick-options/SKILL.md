# /auto-pick-options — 自己排順位、自己做，不回頭問

當使用者觸發 `/auto-pick-options`（或說出上方描述裡那類「你自己決定 / 依你規劃順序做 / 不要問我」的話），代表**方向已授權給你**。你在規劃階段列出的多個選項，不要再用「要 A 還是 B？」把球丟回去；改成由你依自己的判斷排出優先順位，然後直接執行。

## 核心決策：全做 vs. 只做最佳

先判斷這批選項彼此的關係，決定執行策略：

1. **互補、可疊加 → 依序實作全部**。當多個選項各自解決不同面向、彼此不衝突、合起來才完整（例如「先補測試、再修 bug、再更新文件」），就依你排的優先順位**一個一個做完**，不要只挑一個交差。
2. **互斥、二選一 → 只實作最佳單一選項**。當選項是同一件事的不同做法、選了 A 就不會選 B（例如「用 library X 還是自己手刻」「重構成 class 還是保持 function」），就挑你判斷的**最佳單一選項**落地，其餘在收尾時一句話說明為何沒選。
3. **混合情況**：把可疊加的先分成一組依序做完，互斥的部分各自挑最佳；不確定歸類時，**偏向「多做、可疊加」而非「少做」**——寧可把能一起做完的都做掉。

## 排優先順位的準則

決定順序（或決定哪個是「最佳」）時，用這組準則權衡，衝突時由上而下：

1. **正確性 / 安全性**：修 bug、補缺失的驗證、避免資料損毀，永遠排最前。
2. **使用者真正意圖**：最貼近使用者原始需求、最快讓他看到可用結果的先做。
3. **相依順序**：被別的選項依賴的（前置條件）先做；會被後續覆寫的往後放。
4. **投入產出比**：低成本高效益的先做；高風險、大範圍、難回滾的往後排並在進度檔標註。
5. **可逆性**：可逆的先做；不可逆或影響超出 working directory 的（見下方「強制停下」）先停下確認。

## 執行守則

1. **不要用 AskUserQuestion 把選擇丟回去**。既然觸發本 skill，選項排序與取捨就是你的職責。在動手前，**先用一小段文字講清楚**：你判定這批選項屬於「全做」還是「二選一」、排出的順位、以及理由（一兩句即可，不要寫成長篇）。然後直接開始做。
2. **依序推進、逐項交付**。多選項全做時，一項一項完成，別半途跳走；每完成一項就在回應裡標記進度（`[1/3] 完成 …`）。做較大任務時比照 `/safe-yolo`：拆 milestone、每個 milestone `git commit`（`Mn: <繁體中文短描述>`，subject ≤ 72 字）、必要時開 `docs/progress-<slug>.md`。
3. **把假設寫下來**。凡是你替使用者做的判斷（為何這個順序、為何選這個、放棄了什麼），寫進回應或進度檔，讓使用者事後能一眼看懂並在不同意時翻盤。

## 強制停下條件（只有這些才回頭問）

本 skill 授權的是「在你已列出的選項之間自主取捨」，**不是**授權做任何不可逆或超範圍的事。遇到下列情況仍要停下來確認，不可自行決定：

- 即將執行不可逆且影響範圍超出 working directory 的操作（`git push --force`、刪 remote branch、發 PR、寄訊息 / Slack、刪檔覆蓋既有內容等）。
- 觸碰到 `.claude/settings.json` 的 `permissions.deny` 範圍。
- 選項之間的差異牽涉**使用者的重大偏好或成本承諾**（例如選哪個雲服務會產生費用、改動 API 對外契約、切換主要框架），且選錯後難以回滾——這種歧義超出「技術取捨」，要讓使用者拍板。
- 所有選項你都判斷有明顯風險或都不理想時：不要硬選一個，回報你的疑慮並請使用者定奪。

## 注意事項

- **單一職責**：本 skill 只改變「多選項時的決策行為」——由你自主排序並執行，取代「問使用者選哪個」。它**不**負責硬性不停迴圈（那是 `/autopilot`）、不負責 milestone 自動化框架本身（那是 `/safe-yolo`，本 skill 只在做大任務時沿用其 commit / 進度慣例）、不改任何設定或 hook。
- **與 /autopilot、/safe-yolo 的關係**：可疊加。`/safe-yolo`、`/autopilot` 規範「不停下來、持續推進」；本 skill 補上「當推進途中冒出多個選項該怎麼自己選」。三者可同時生效。
- **不是無腦全選**：核心價值在「正確地分類 + 排序」，而不是把每個選項都做一遍。互斥選項硬全做只會製造矛盾——該二選一時就果斷二選一。
- **透明優先**：自主決策的前提是事後可被使用者檢視與推翻，所以「講清楚你怎麼選的」與「動手做」同等重要，不可只做不說。
